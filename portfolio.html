"Juego" Website



While using Agile methodology, my team and I developed a website like "Tinder" for gamers. This was a small project we worked that allowed me to use google firebase, SQL, React, CSS, APIs (such as GoogleOAuth), and JavaScript. The user creates an account and profile which adds them to the database and allows other users to see their card with their gaming information. If they swipe right, they can visit their profile later to add them. Swiping left moves to the next person in the stack.

HACK Assembler /
Disassembler



Using C++, I built a HACK assembler and disassembler. The assembler will read in the symbolic HACK assembly language (from input files) and convert it into HACK machine code, creating an output file. In addition, the disassembler does the opposite and reads a 15-bit binary number and converts it back into the symbolic assembly language, creating an output file.

Greedy Algorithm - Elevation Levels



This takes in elevation points of data from provided .dat files (or the user can make their own), stores them in a vector, converts them into shades of gray, displays the paths taken (in red), and clarifies the "best" path (in green) utilizing the greedy algorithm.

Bulls and Cows Game



This is a game where the user guesses a randomly generated code (only 3, 4, or 5 digits long). Each number is unique. When the user inputs their guess, they get in return the number of bulls and cows within their answer. A cow means one of the numbers are correct, but it is in the wrong spot. A bull means one of the numbers is present in the code and is in the correct spot.
